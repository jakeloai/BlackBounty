# 🛡️ BlackBounty
**Hunter : Jake**

**BlackBounty** is an enterprise-grade, automated reconnaissance and vulnerability scanning pipeline. It is specifically engineered for **large-scale bug hunting** where bypassing WAFs, avoiding IP bans, and receiving real-time alerts are critical to success.

---

## ⚙️ How It Works

BlackBounty orchestrates a multi-stage security workflow while maintaining a "Zero-Exposure" profile for your local IP:

1.  **Dynamic Proxy Harvesting**: Automatically scrapes 5,000+ public HTTP, SOCKS4, and SOCKS5 proxies from high-quality GitHub repositories.
2.  **Health Check Engine**: Performs high-concurrency validation (50 threads) to ensure only "Live" proxies are used.
3.  **Circuit-Breaker Logic**: Automatically generates a `proxychains4` configuration using `dynamic_chain` mode—if one proxy fails, the traffic instantly hops to the next.
4.  **Multi-Stage Recon**:
    * **Subfinder**: Deep subdomain discovery across multiple sources.
    * **HTTPX**: Probes for live web servers using native proxy rotation for maximum speed.
5.  **Intelligent Scanning**: Executes `Nuclei` templates across the attack surface.
6.  **Real-Time Alerts**: Pings your **Discord** via Webhooks the second a vulnerability is detected.

## 🚀 Installation & Setup

### 1. Clone and Prepare
Ensure you are running on a Linux-based system (Kali Linux or Ubuntu recommended).

```bash
chmod +x install.sh BlackBounty.sh
sudo ./install.sh
```

### 2. Discord Integration
To receive real-time alerts, you will need a Discord Webhook.
* Go to **Discord** > **Server Settings** > **Integrations** > **Webhooks**.
* Create a New Webhook and copy the **Webhook URL**.
* The **Webhook ID** is the first long string of numbers in that URL.

---

## 🎯 Usage

To start a new hunting session, simply provide a file containing your target root domains:

**targets.txt**
```text
example.com
target-site.net
```

**Run the command:**
```bash
blackbounty targets.txt
```

> **Note:** On the first run, the script will interactively guide you to set up your Discord credentials and save them to `~/.config/notify/provider-config.yaml`.

---

## 📂 Output Directory (`bounty_output/`)

The script organizes all data into a dedicated folder for each session:

| File | Description |
| :--- | :--- |
| `raw_proxies.txt` | The complete list of scraped proxies. |
| `valid_proxies.txt` | Proxies that passed the 5s latency health check. |
| `01_subs_raw.txt` | All unique subdomains discovered. |
| `02_alive_hosts.txt` | Filtered list of hosts responding to HTTP/HTTPS. |
| `03_vulns.txt` | The final vulnerability report generated by Nuclei. |

---

## 🛠 Dependencies
BlackBounty automatically installs and configures:
* `Proxychains4` (Traffic routing)
* `Subfinder` (Passive recon)
* `HTTPX` (Active probing)
* `Nuclei` (Vulnerability scanning)
* `Notify` (Discord/Slack integration)

* **Security**: Never share your `provider-config.yaml` or Webhook URLs publicly.

```bash
screen -S hunting

```
